# Zanzibar Land Title Registration System

A full-stack web application for land title registration in Zanzibar.

## Stack

| Layer | Technology |
|-------|-----------|
| Backend API | Django 5 + Django REST Framework |
| Auth | JWT via `djangorestframework-simplejwt` |
| Database | PostgreSQL (Render managed) |
| Frontend | React 18 + Vite |
| UI Components | shadcn/ui + Radix UI + Tailwind CSS |
| Deployment | Render (backend web service + static site) |

---

## Features

- **Dashboard** — live stats: parcels, active deeds, owners, pending applications; application pipeline chart
- **Land Parcels** — CRUD, filter by district / land use / status, 10 Zanzibar districts
- **Title Deeds** — register deeds linked to parcels + owners, automatic parcel status update
- **Applications** — submit new registration / transfer / subdivision / correction applications; officer review workflow
- **Owners** — owner registry with national ID, contact info, active deed count
- **JWT Auth** — 8-hour access token, 7-day refresh token, auto-refresh on 401

## Local Development

### Backend

```bash
cd backend
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py create_default_admin   # creates admin / Admin@Zanzibar2024!
python manage.py runserver
```

API runs at `http://localhost:8000`

### Frontend

```bash
cd frontend
pnpm install
# Vite proxies /api → localhost:8000 automatically
pnpm dev
```

App runs at `http://localhost:5173`

## Deploy to Render

### 1. Push to GitHub

```bash
git add .
git commit -m "Initial commit: Zanzibar Land Title Registration System"
git remote add origin <your-github-repo-url>
git push -u origin main
```

### 2. Create Render services

Go to [render.com](https://render.com) → **New → Blueprint** → connect your repo.
Render will read `render.yaml` and create:
- `landreg-db` — PostgreSQL database (free tier)
- `landreg-api` — Python web service
- `landreg-frontend` — Static site

### 3. Set environment variables after first deploy

In the Render dashboard for **landreg-api**:
| Variable | Value |
|----------|-------|
| `CORS_ALLOWED_ORIGINS` | `https://landreg-frontend.onrender.com` |

In the Render dashboard for **landreg-frontend**:
| Variable | Value |
|----------|-------|
| `VITE_API_URL` | `https://landreg-api.onrender.com/api` |

Trigger a redeploy for both services after setting these.

### 4. Default admin credentials

The build script auto-creates an admin account. Check the `ADMIN_PASSWORD` generated value in
the `landreg-api` service environment tab on Render — it's auto-generated by Render.

You can override it by setting `ADMIN_USERNAME`, `ADMIN_PASSWORD`, `ADMIN_EMAIL` env vars.

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/token/` | Login (returns access + refresh tokens) |
| POST | `/api/token/refresh/` | Refresh access token |
| GET | `/api/me/` | Current user profile |
| GET | `/api/stats/` | Dashboard statistics |
| CRUD | `/api/parcels/` | Land parcels |
| CRUD | `/api/owners/` | Land owners |
| CRUD | `/api/deeds/` | Title deeds |
| CRUD | `/api/applications/` | Applications |
| PATCH | `/api/applications/{id}/review/` | Officer review action |

All endpoints require `Authorization: Bearer <token>` except `/api/token/`.
